# 🎯 AI Security Setup Wizard - Руководство

## 📋 Описание

**Setup Wizard** - это интерактивный мастер настройки, который:

1. **Анализирует** вашу систему (открытые порты, службы)
2. **Задаёт вопросы** о назначении сервера
3. **Предлагает** оптимальные настройки
4. **Автоматически применяет** конфигурацию

---

## 🚀 Быстрый старт

```bash
cd ~/security-monitor
./setup-wizard.sh
```

---

## 📊 Этапы настройки

### Шаг 1: Анализ системы

**Что делает:**
- Сканирует открытые порты
- Определяет активные службы
- Показывает текущую конфигурацию

**Пример вывода:**
```
Открытые порты:
   • Порт 22 (SSH)
   • Порт 80 (HTTP)
   • Порт 443 (HTTPS)

Всего открытых портов: 3
✅ SSH активен (порт 22)
✅ Веб-сервер обнаружен
```

---

### Шаг 2: Тип сервера

**Варианты:**

| № | Тип | Описание |
|---|-----|----------|
| 1 | 🌐 Веб-сервер | Сайт, блог, магазин |
| 2 | 🔧 Сервер приложений | Node.js, Python, Java |
| 3 | 💾 База данных | MySQL, PostgreSQL, MongoDB |
| 4 | 📁 Файловый сервер | FTP, SFTP, Nextcloud |
| 5 | 🎮 Игровой сервер | Minecraft, CS:GO |
| 6 | 📧 Почтовый сервер | SMTP, IMAP, POP3 |
| 7 | 🔬 Разработка | Тестовый сервер |
| 8 | ⚙️ Другое | Свой вариант |

---

### Шаг 3: Уровень безопасности

**Три режима:**

#### 🟢 МЯГКИЙ
- Минимум блокировок
- Для разработки
- Меньше ложных срабатываний

#### 🟡 СТАНДАРТНЫЙ (рекомендуется)
- Баланс защиты
- Для продакшена
- Оптимальные настройки

#### 🔴 АГРЕССИВНЫЙ
- Максимальная защита
- Быстрые блокировки
- Возможны ложные срабатывания

---

### Шаг 4: Настройка портов

**Автоматические рекомендации:**

| Тип сервера | Рекомендуемые порты |
|-------------|---------------------|
| Веб-сервер | 22, 80, 443 |
| Сервер приложений | 22, 8080, 443 |
| База данных | 22 (порт БД закрыть!) |
| Игровой сервер | 22, 25565 |

**Можно добавить свои порты.**

---

### Шаг 5: Белый список

**Что делает:**
- Добавляет доверенные IP
- Защищает от блокировки
- Автоматически сохраняет

**Пример:**
```
Введите IP адреса для белого списка:
IP #1: 192.168.1.100
✅ Добавлен: 192.168.1.100

IP #2: 10.0.0.50
✅ Добавлен: 10.0.0.50

IP #3: [Enter]
Добавлено IP в whitelist: 2
```

---

### Шаг 6: Мгновенный бан

**Настройка порога:**

| Уровень | Порог | Описание |
|---------|-------|----------|
| Мягкий | 20 | Бан при 20+ попыток |
| Стандартный | 15 | Бан при 15+ попыток |
| Агрессивный | 10 | Бан при 10+ попыток |

**Можно задать своё значение.**

---

### Шаг 7: Telegram уведомления

**Типы уведомлений:**
- 🟠 Средний уровень угрозы
- 🔴 Высокий уровень угрозы
- 🚨 Критический уровень
- ⏰ Ежечасные отчёты

---

### Шаг 8: Применение настроек

**Показывает итоговую конфигурацию:**
```
📊 ИТОГОВАЯ КОНФИГУРАЦИЯ:

Тип сервера: WEB_SERVER
Уровень безопасности: СТАНДАРТНЫЙ (2)
Мгновенный бан: true (порог: 15)
Whitelist IP: 192.168.1.100,10.0.0.50
Уведомления: Включены

Применить настройки? (y/n):
```

## 🛡️ Резервное копирование

**Перед применением настроек:**
- Автоматически создаётся резервная копия
- Сохраняется с датой и временем
- Путь: `ai_config.conf.backup.YYYYMMDD_HHMMSS`

**Восстановление:**
```bash
cp ai_config.conf.backup.20260309_170000 ai_config.conf
pkill -f ai_security_v3 && nohup ./ai_security_v3.sh > /dev/null 2>&1 &
```

## ⚠️ Важные замечания

### 1. Тип сервера

- **Выбирайте правильно** - от этого зависят рекомендации
- **База данных** - не открывайте порт наружу!
- **Игровой сервер** - готовьтесь к DDoS

### 2. Уровень безопасности

- **Мягкий** - для разработки
- **Стандартный** - для большинства
- **Агрессивный** - при активных атаках

### 3. Whitelist

- **Добавляйте постоянные IP**
- **Не добавляйте динамические**
- **Проверяйте список**

### 4. Порты

- **Открывайте только нужные**
- **Закрывайте лишние**
- **Используйте firewall**

## 🆘 Troubleshooting

### Мастер не запускается:

```bash
# Проверьте права
chmod +x ~/security-monitor/setup-wizard.sh

# Запустите с полным путём
/home/larson/security-monitor/setup-wizard.sh
```

### Настройки не применились:

```bash
# Проверьте конфиг
cat ~/security-monitor/ai_config.conf

# Перезапустите службу
pkill -f ai_security_v3
nohup ~/security-monitor/ai_security_v3.sh > /dev/null 2>&1 &
```

### Whitelist не работает:

```bash
# Проверьте файл
cat ~/security-monitor/whitelist.txt

# Перезагрузите систему
pkill -f ai_security_v3
nohup ~/security-monitor/ai_security_v3.sh > /dev/null 2>&1 &
```
